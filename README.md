# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 투자자들을 위한 "주식 평균단가 시뮬레이터"


## 2. 시나리오 (5~10줄)
* 저는 평소에 주식에 관심이 많아서 주식에 관련된 프로그램을 만들고 싶어서 어떤 프로그램이 실제로 쓸수 있고 유용할까 생각해보다가 "주식 평균단가 시뮬레이터" 라는 프로그램을 만들기로 결정했습니다
* 이 프로그램은 주식 투자자가 여러 번에 걸쳐 매수한 종목의 평균 단가를 계산하고, 향후 추가 매수(물타기) 시의 변화를 예측하는 종합 자산 관리 도구입니다.
* 사용자는 보유 중인 주식의 매수가와 수량을 각각 입력하며, 시스템은 이를 바탕으로 총 투자 금액과 가중 평균을 적용한 평균 단가를 산출합니다.
* 추가로 현재 주가를 입력하면 실시간 수익률을 계산해주고 특정 목표 평단가에 도달하기 위해 추가로 매수해야 할 수량을 제안하는 기능을 포함합니다
* 사용자는 메뉴판을 통해 정보를 수정하거나 초기화할 수 있고, 보유한 주식이 여러개인 투자자들을 위해서 주식들 각각의 평균단가와 실시간 수익률을 저장할 수 있게 만들 것입니다.
* 또한 여러 번의 분할 매수 기록을 리스트로 관리하여 최종 투자 현황을 한눈에 조회할 수 있도록 할 것입니다.
* 그리고 구현이 된다면 그래프를 이용하여 전 일/달/년 대비 수익률을 조회할 수 있도록 시스템을 구현하고 싶습니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 매수 기록 입력: 종목별 매수 가격과 수량을 반복해서 입력받는 기능
2. 평균 단가 실시간 계산: 누적된 데이터를 바탕으로 평균 단가 및 총액 산출
3. 수익률 분석: 현재 주가 입력 시 평단가 대비 수익률(%) 및 손익 금액 출력
4. 물타기 시뮬레이션: 목표 평단가를 입력하면 현재가 기준 필요한 매수 수량 계산
5. 투자 리포트 출력: 현재까지의 모든 매수 내역과 최종 지표를 표 형태로 출력
6. 현재 까지의 수익률을 일/달/년 으로 나누어 그래프 형태로 출력
7. 데이터 초기화 및 종료: 입력된 내역을 지우거나 프로그램을 안전하게 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 시나리오 기획
    * "주식 평단가 계산기" 사용자로부터 종목명(문자열), 현재 보유 수량(정수), 평균 매수가(정수), 현재가(정수)를 입력받아 총 투자 금액과 현재 수익률을 계산하여 출력합니다.
    * 종목명(문자열), 수량(정수), 금액(정수/실수) 등 다양한 자료형을 입력받는 화면을 구현했습니다.
    * 사칙연산을 활용해 '총 매수 금액'과 '수익률' 산출 로직을 구현했습니다.
    
### **🤖 AI 파트너십 과정**

 1. **내용 1 : 주식 평균단가 계산기 세부 내용 구체화
    * **프롬프트 요약:** 나는 평소에 주식에 관심이 있어서 주식에 관련된 프로그램을 만들고 싶어서 내가 생각한 "나의 주식 평균단가 계산기" 라는 프로그램을 어떻게 구상하고 출력해야할지 요청함.
    * **적용 내용:** AI가 제안한 여러 아이디어 중 평균단가를 계산해주는 것과 내가 몇주를 더 매수하면 내가 목표하는 평균단가에 도달할지, 그리고 현재 주가를 기준으로 수익률이 몇%인지를 계산해주는 시스템을 추가해줘서 나의 아이디어가 더욱 풍부해짐. 그리고 입력한 정보를 바탕으로 출력된 데이터들을 어떤식으로 화면에 보이면 좋을지를 제시해줌.
      
 2. **내용 2 : 로드맵 도출**
    * **프롬프트 요약:** 1차 과제 필수 요건(변수 5개 이상, 자료형 3개 이상 사용)을 충족하면서 주식 데이터를 효율적으로 처리하기 위한 변수 설계 및 확장 로드맵 논의.
    * **적용 내용:** 변수 설계과정에서 데이터 특성에 따라 종목명(str), 보유 수량(int), 평균 매수가(int), 현재가(int), 수익률(float) 등 5개 이상의 변수를 설정하여 구조적 완성도를 높임.
                     단순 입출력에 그치지 않고, 입력(1차) → 수익률에 따른 투자 상태 판별(2차) → 반복문을 통한 다회차 매수(3차) → 리스트를 활용한 포트폴리오 관리(4차)로 이어지는 논리적 개발 로드맵을 수립함.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
* 이번 과제는 개인 투자 포트폴리오 관리 및 투자 등급 진단 시스템 구현으로, 사용자로부터 주식 데이터를 입력받아 리스트에 저장하고, 산술 연산을 통해 투자 성과를 도출하는 시스템을 구축하는 것입니다. 특히 if-elif 조건문과 and/or 논리 연산자를 활용하여 수익률에 따른 투자 등급과 맞춤형 특별 칭호를 부여하는 지능형 로직을 완성했습니다.
* input()으로 투자자 이름, 목표 수익률, 종목명, 매수 단가를 입력받아 각각 변수와 리스트(stock_list, price_list)에 저장합니다.
* for 반복문과 continue 제어문을 도입하여 데이터 입력의 유연성을 확보하고, 0 이하의 잘못된 값이 입력될 경우 리스트 저장을 건너뛰는 예외 처리 로직을 구현했습니다.
* 리스트 내장 함수(sum(), len())를 활용하여 평균 매수 단가를 산출하고, 이를 현재가와 비교하여 실시간 수익률을 계산합니다.
* if-elif-else를 사용하여 수익률에 따라 **S등급(전설의 가치투자자)부터 F등급(위험 상태)**까지 5단계 등급 판정 로직을 추가했습니다.
* 비교 연산자를 활용하여 일정 수익률 초과, 미달 시 '오늘 저녁은 원하는걸 드세요!' , '손실중 대책을 강구하세요!' 등의 축하, 경고 메세지를 나타내도록 했습니다.

### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** 목표수익률, 종목명, 평균단가, 총투자액등과같은 용어들의 변수이름을 어떻게 설정하면 좋을까?
    * **적용 내용:** 투자자 이름을 user_name 종목명을 stock_name 현재가격을 current_price로 설정하고 AI가 제시한 변수이름을 변형하여 목표수익률을 goal_price , 평균 단가를 average_price로 설정함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "주식 평단가 계산기를 만들고 있는데 단순히 계산만 하는 게 아니라, 리스트에 데이터를 저장하고 수익률에 따라 투자 등급을 매기는 프로그램으로 만들고 싶어. 1학년 수준에서 사용할 수 있는 조건문과 논리 연산자 활용 아이디어를 줘"
    * **적용 내용:** AI가 제안한 **'수익률 기반 5단계 등급 판정'**과 **'수익률 AND 투자금액'**을 결합한 복합 조건문을 실제 코드에 반영함. 특히, 데이터 입력 단계에서 0 이하의 값이 들어올 경우 프로그램 오류를 방지하기 위해 continue를 활용한 데이터 필터링 로직을 추가하여 완성도를 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** 잘못된 가격(0원 이하)을 입력했을 때 continue를 사용했는데 total_investment += s_price 코드 뒤에 배치해서 잘못된 가격까지 포함되는 오류 발생
    * **해결:** continue 문을 데이터를 저장하고 합산하는 코드보다 위쪽에 배치하여 조건에 맞지 않는 데이터는 넘어가도록 순서를 바꿈
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** avg_price = int(input("매수 단가를 입력하세요: ") 에서 괄호를 하나 빼먹어서 SyntaxError가 발생
    * **해결:** 파이썬의 모든 조건문과 반복문 끝에는 항상 땡땡(`:`)이 들어가야 한다는 규칙을 다시한번 가슴속에 새기고 괄호를 하나 추가함
